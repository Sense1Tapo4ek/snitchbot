Frozen dataclass that groups the seven detectors snitchbot runs against per-process vitals (own metrics) and aggregate vitals (process + recursive children). Every field defaults to its spec-default instance — passing `AnomalyConfig()` is the same as passing nothing. Set a slot to `None` to disable **that one** detector; use [`AnomalyConfig.all_disabled()`](#fields) to turn everything off. `init()` accepts `AnomalyConfig`, `True` (defaults), `False` (all off) or `None` (defaults) in its `anomaly` argument.

Defaults match the source — read them verbatim below.

## Fields

| Name | Type | Description | Default |
|---|---|---|---|
| `rss` | [`RssAnomalyConfig`](./rss-anomaly-config) `\| None` | Memory detector (own process only). | `RssAnomalyConfig()` |
| `cpu` | [`CpuAnomalyConfig`](./cpu-anomaly-config) `\| None` | CPU% detector (own process only). | `CpuAnomalyConfig()` |
| `fds` | [`FdAnomalyConfig`](./fd-anomaly-config) `\| None` | File-descriptor detector. | `FdAnomalyConfig()` |
| `threads` | [`ThreadAnomalyConfig`](./thread-anomaly-config) `\| None` | Thread-count detector. | `ThreadAnomalyConfig()` |
| `watchdog` | [`WatchdogConfig`](./watchdog-config) `\| None` | Asyncio event-loop watchdog. | `WatchdogConfig()` |
| `total_rss` | [`RssAnomalyConfig`](./rss-anomaly-config) `\| None` | Aggregate memory detector (process + all children). | `None` (disabled) |
| `total_cpu` | [`CpuAnomalyConfig`](./cpu-anomaly-config) `\| None` | Aggregate CPU% detector (process + all children). | `None` (disabled) |

Class methods:

- `AnomalyConfig.defaults()` — equivalent to `AnomalyConfig()`.
- `AnomalyConfig.all_disabled()` — every slot set to `None`.
- `AnomalyConfig.from_dict(data)` — inverse of `.resolve()`; used by the sidecar on handshake.
- `.resolve()` — serialise to the msgpack-friendly wire dict.
- `.max_history_seconds()` — longest `baseline_duration` across enabled detectors (drives the vitals deque size).

## Example

```python
import snitchbot
from snitchbot import AnomalyConfig, RssAnomalyConfig, WatchdogConfig

snitchbot.init(
    "orders-api",
    anomaly=AnomalyConfig(
        rss=RssAnomalyConfig(max_mb=600, spike_ratio=2.0),
        watchdog=WatchdogConfig(threshold_ms=300),
        # cpu / fds / threads keep their defaults
    ),
)

# Disable everything:
snitchbot.init("batch-job", anomaly=False)
```

## Notes

`None` on a detector slot disables only that detector — the rest keep their defaults. Bad construction raises `InvalidAnomalyConfigError` at dataclass init (before `snitchbot.init` touches any state).
